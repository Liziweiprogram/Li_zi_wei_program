Build-Blog(in Github)

分享

声明：此文在论如何搭建一个静态博客的基础上建立的，并非原创

搭建博客材料：git Node.js Github账号(邮箱验证过)

注意：由于Git官网下载需要8小时(速度大约1000B/s,Git大小17Mb左右)，所以推荐在电脑管家——软件管理——搜索Git 下载(大约1分钟) 或 在百度搜索百度网盘下载(大约10分钟(非会员))

P.S. 本人电脑性能
  • 处理器：Intel(R) Code(TM) i7-8565UCPU @ 1.80GHz 1.99Ghz
  • 内存条：8.00GB
  • 系统：Windows x64

Git-Bash的初始使用

Git Node.js下载完成后打开安装Git文件夹中的Git-Bash.exe

注意：打开Git-Bash的时候会有报错信息 直接忽略按I see即可

检测Node.js安装

在Git-Bash界面输入node -v 和 npm -v检测Node.js安装

提示版本号信息则说明安装成功

hexo登场

参照git-bash的初始使用打开git-bash并输入npm install hexo-cli -g

同样，你可以输入hexo -v检测安装hexo是否成功（成功会跳出版本号信息）

开始搭建博客

首先新建文件夹，这个文件夹是你未来的博客文件夹，所以命名可以命名为blog

在这个空文件夹中右键，会看到一个Git-Bash There，点开

输入hexo init

然后祈祷一会儿

当你的文件结构长这样时就好了

还是在这个文件夹里输入hexo s(注意不要点到子文件夹里Git-Bash了)

打开提示最后一行的网址(INFO Hexo is runnming at http://localhost:4000 . Press Ctrl + C to stop)

结束预览请按Ctrl + C 或者关闭Git-Bash

如果打不开网页，那么十有八九是端口被占用了，只要输入hexo s -p 端口号，然后打开http://localhost:端口号/就行了(端口号随你填)。

论如何搭建一个静态博客到这里就结束了，然后静态博客搭建还没有结束。。。

Github的仓库

在右上角的+号打开New repository

输入Repository name(注意这个名字是你Github用户名.github.io(比如我是liziweiprogram.github.io))

其他默认设置就Create Repository就行了

仓库建好不要动

Github设置SSH key

然后打开Git-Bash输入ls -al ~/.ssh

(接下来所有输入都在Git-Bash内完成)

如果提示No such file or directory则正常

然后输入ssh-keygen -t rsa -C "github注册邮箱" (比如我就输入ssh-keygen -t rsa -C “3155744679@qq.com”)

输完了之后会提示你设置密码(不想设置可以直接回车(我就没设置))

继续回车(如果没有设置密码) 设置密码则再次输入原来的密码

然后会弹出来一大堆东西(直接忽略)

ssh-agent -s

再输入ssh-add ~/.ssh/id_rsa

然后eval 'ssh-agent -s'和ssh-add

如果没有FAFAL或者Error打开打开C:\Users\username

里面如果有文件夹.ssh,且文件夹里有id_rsa就成功了

输入cat ~/.ssh/id_rsa.pub

会弹出一份ssh-rsa开头，你github注册邮箱结尾的SSH key

整体复制(全选这个ssh-rsa开头，你github注册邮箱结尾的SSH key,右键copy)

按上图进入SSH key界面

点New SSH key

把刚才那一长串SSH Key复制到这个界面的Key里面

Title随便填qwq

之后(第一次)会让你输入Github密码

如果输完成之后有一个SSH key就成功啦

输入ssh -T git@github.com测试

这里会弹出来叫你输入yes no请输入yes

然后没有问题就会有一个Warning

_config.yml(最后一步了)

找到_config.yml的#side 部分

再看#URL

最后deploy

完成了！

如果有什么疑问的话,我把我的_config.yml文件拿出来

最后一步的最后一步——填信息

git config --global user.email "(你的github注册邮箱)"

git config --global user.name "(你的github用户名)"

(因为我这里没弄前面hexo d完全没用)

看看你的博客吧

在你的blog文件夹Git-Bash(方法在前文提到过了)

输入hexo g &&hexo d

接下来进入你Github博客的仓库code

文件为上面格式则正常

然后用浏览器打开(你的博客仓库名字).github.io(比如我是liziweiprogram.github.io)

如果前面都正确完成的话,打开就能看你的博客了!(现在只有Hello World哦)

你知道吗,你的博客已经搭建完毕了!
