分享

mkdocs安装指南

网上材料好的差的都有 所以来写一篇

搭建博客材料：git Node.js Github账号(邮箱验证过) pip 你的github博客(否则请回去看Building-Blog(in Github))

还是这些

mkdocs 是什么?

如果你不喜欢hexo的风格(每个文件都是/year/month/day/name)的,mkdocs会是一个搭建博客的好选择(文件路径就是直接/mkdocsname/name)

但是mkdocs必须在你已经有xxx.github.io仓库了才可以 而且搭建完成后,路径(进入mkdocs)一定是xxx.github.io/mkdocsname

当然你也可以选择CNAME,但那是另外一回事了

开始搭建mkdocs

P.S 名称为xxxxxname的都是你自己的项目名称 请不要直接复制

首先,先在github上弄一个叫”Yourmkdocsname”(里面是你项目名字)的仓库(初始化+Readme)

然后 git clone git@github.com:yourgithubname/yourmkdocsname.git(自己填好你的yourgithubname和yourmkdocsname)

接下来 在gitclone下来的文件夹里mkdocs new yourmkdocsname

然后你看到的应该是这样的:

doc文件夹里还有一份index.md

这时候你是不是发现你的路径至少是> yourmkdocsname > yourmkdocsname?

把刚刚弄的东西搬出来(也就是把路径缩短成> youemkdocsname)

mkdocs.yml

首先给大家看看示范mkdocs.yml(也就是我的)

P.S nav里面的about.md等 都是自己的md文件

另外 mkdocs还有其他主题 可以自己去https://markdown-docs-zh.readthedocs.io看theme

update

如果你的mkdocs.yml以及md都弄好了 接下来可以预览了

mkdocs serve

如果显示正常 成功了一半(没成功回去看.yml是不是炸了)

效果差不多是hexo s

(Ctrl+C结束预览)

接下来

mkdocs build

效果差不多是hexo g

最后

mkdocs gh-deploy

效果差不多是hexo d

但是我在这里出了问题——开始弄的时候文件夹不是git clone下来的(所以如果按前面弄应该没问题了)
